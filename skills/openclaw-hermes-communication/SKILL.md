# openclaw-hermes-communication

## 触发条件

当 Hermes Agent（小马）需要与 OpenClaw 小龙协作时激活此 skill。

典型场景：
- 用户向 Hermes 发送股票查询请求
- 用户向 Hermes 发送需要 OpenClaw 小龙处理的通用任务
- Hermes 需要调用小龙的专业能力（股票分析、图片生成、代码编写等）

---

## 核心流程

### 第1步：立即回复用户

收到任务后，**立即**发送确认消息（不等待任何工具）：

```
收到，正在转交给小龙处理...
```

### 第2步：判断任务类型

根据任务内容判断 `type` 参数：

| 任务关键词 | type 值 | 说明 |
|-----------|---------|------|
| 股票、股价、涨跌、开户、持仓、K线 | `stock` | 股票查询任务 |
| 代码、编程、调试、函数 | `code` | 代码任务 |
| 图片、画图、生成图片 | `image` | 图片生成任务 |
| 其他所有任务 | `general` | 通用任务 |

### 第3步：下发任务给小龙

使用 curl 向 OpenClaw 小龙发送 POST 请求：

```bash
curl -X POST http://192.168.1.100:8080/dispatch \
  -H "Content-Type: application/json" \
  -d '{
    "task": "任务内容",
    "type": "stock|general",
    "timeout": 900
  }'
```

**参数说明：**
- `task`：用户的原始任务内容（字符串）
- `type`：`stock`=股票查询任务，`general`=通用任务（默认）
- `timeout`：超时时间（秒），默认 900s（15分钟）

### 第4步：处理返回结果

**成功情况**：小龙返回了实际执行结果
- 解析返回内容
- 展示给用户

**降级机制**：小龙返回的内容与下发任务内容**完全相同**（未实际执行）
- 判断方式：对比返回的 task 字段与下发的 task 字段是否完全一致
- 触发条件：返回内容 == 下发任务内容（字符串相等）
- 降级策略：使用 Hermes 自己的通用模型自行处理任务
- 处理方式：直接用 AI 能力回答用户，不再次调用小龙

### 第5步：展示最终结果

将最终结果以清晰格式展示给用户。

## 完整示例

```bash
# 示例：用户问"贵州茅台今天股价多少"
TASK="贵州茅台今天股价多少"

# 1. 立即回复
echo "收到，正在转交给小龙处理..."

# 2. 判断类型 → stock
# 3. 下发任务
curl -X POST http://192.168.1.100:8080/dispatch \
  -H "Content-Type: application/json" \
  -d "{\"task\": \"$TASK\", \"type\": \"stock\", \"timeout\": 900}"

# 4. 收到结果后处理
# 5. 展示给用户
```

---

## 注意事项

1. **立即回复**：收到任务后必须第一时间回复"收到"，不要等待小龙返回
2. **类型判断**：准确判断 `type` 可以让小龙更快路由到专业小弟
3. **降级触发**：只有当小龙返回内容与下发内容完全一致时才触发降级
4. **超时设置**：股票查询建议 timeout=900，通用任务可用默认值
5. **错误处理**：
   - 如果 curl 请求失败，记录错误并告知用户
   - 如果小龙无响应，等待 timeout 后降级处理
6. **任务ID传递**：如果原始请求包含 task_id，需要在回调时使用

## 小龙虾团队协作架构

```
用户 → Hermes Agent
           ↓
    判断任务类型 + 立即回复
           ↓
    POST /dispatch 到小龙(8080)
           ↓
    小龙 spawn 专业的「小弟」Agent
           ↓
    小弟通过 HTTP API 调用 Python Sub-Agent 获取数据
           ↓
    AI 分析 + 返回结果
           ↓
    Hermes 展示结果给用户
           ↓
    如小龙未执行 → Hermes 降级自行处理
```
